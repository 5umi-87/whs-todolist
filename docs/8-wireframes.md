## 1. 화면 개요

### 1.1 화면 목록 및 라우트

| 화면명 | 라우트 | 인증 요구 | 우선순위 | 설명 |
|--------|--------|----------|----------|------|
| 로그인 | `/login` | No | P0 | 이메일/비밀번호 로그인 |
| 회원가입 | `/register` | No | P0 | 신규 사용자 등록 |
| 할일 목록 (메인) | `/` | Yes | P0 | 할일 조회 및 관리 메인 화면 |
| 휴지통 | `/trash` | Yes | P0 | 삭제된 할일 관리 |
| 국경일 | `/holidays` | Yes | P0 | 국경일 조회 |
| 프로필 | `/profile` | Yes | P1 | 사용자 정보 조회/수정 |
| 관리자 국경일 관리 | `/admin/holidays` | Yes (Admin) | P1 | 국경일 추가/수정 (관리자 전용) |

### 1.2 화면 플로우 다이어그램

```
[시작]
   |
   v
[로그인 페이지] ──────────> [회원가입 페이지]
   |                              |
   | (인증 성공)                  | (가입 완료)
   v                              v
[할일 목록 메인 페이지] <─────────┘
   |
   ├─> [할일 추가 모달]
   ├─> [할일 수정 모달]
   ├─> [할일 상세 모달]
   ├─> [휴지통 페이지]
   ├─> [국경일 페이지]
   ├─> [프로필 페이지]
   └─> [관리자 국경일 관리 페이지] (관리자만)
```

### 1.3 네비게이션 구조

**데스크톱**:
- Header 고정 (로고, 네비게이션 링크, 사용자 메뉴)
- 좌측 사이드바 (선택 사항): 할일 목록, 휴지통, 국경일

**모바일**:
- Header 간소화 (로고, 햄버거 메뉴)
- 하단 고정 탭 네비게이션 (홈, 휴지통, 국경일, 프로필)

## 2. 공통 레이아웃

### 2.1 Header 컴포넌트

#### 데스크톱 버전
```
┌────────────────────────────────────────────────────────────────────┐
│  [Logo] WHS-TodoList    [할일] [휴지통] [국경일]       [홍길동 ▼] │
└────────────────────────────────────────────────────────────────────┘
```

**컴포넌트 명세**:
- **로고**: 클릭 시 메인 페이지(`/`)로 이동
- **네비게이션 링크**:
  - 할일 (`/`)
  - 휴지통 (`/trash`)
  - 국경일 (`/holidays`)
- **사용자 메뉴 드롭다운**:
  - 사용자 이름 표시
  - 드롭다운 메뉴: [프로필] [로그아웃]
  - 클릭 시 드롭다운 토글

**스타일**:
- 배경: 흰색 (다크모드: #1A1A1A)
- 높이: 64px
- 하단 그림자: shadow-sm
- 고정: sticky top-0
- z-index: 50

#### 모바일 버전
```
┌──────────────────────────────────────┐
│  [☰]  WHS-TodoList        [홍길동]  │
└──────────────────────────────────────┘
```

**햄버거 메뉴 (열림 상태)**:
```
┌──────────────────────────────────────┐
│  [×]  메뉴                           │
├──────────────────────────────────────┤
│  [🏠] 할일 목록                      │
│  [🗑️] 휴지통                         │
│  [📅] 국경일                         │
│  [👤] 프로필                         │
│  [🚪] 로그아웃                       │
└──────────────────────────────────────┘
```

**스타일**:
- 햄버거 아이콘: 좌측 상단
- 사용자 이름: 우측 상단 (아이콘만)
- 메뉴 오버레이: 전체 화면 또는 좌측 슬라이드

  - 옵션: 만료일 (빠른순/늦은순), 생성일 (최신순/오래된순)

**할일 카드 (TodoCard)**:
- **체크박스**: 완료 처리 토글
- **제목**: 글자 크기 lg, 굵게
- **내용**: 글자 크기 sm, 최대 2줄 (말줄임표)
- **날짜**:
  - 아이콘: 📅
  - 형식: "MM-DD" 또는 "MM-DD ~ MM-DD"
- **상태 라벨**:
  - 진행중: 주황색 배지 🕐
  - 완료: 초록색 배지 ✅
  - 만료 임박: 빨간색 경고 ⚠️ D-X일
  - 만료됨: 빨간색 배지 ⏰
- **액션 버튼**:
  - 수정: 아이콘 버튼 (연필)
  - 삭제: 아이콘 버튼 (휴지통)

**플로팅 액션 버튼 (FAB)**:
- 위치: fixed right-6 bottom-6
- 크기: 56x56px (모바일), 64x64px (데스크톱)
- 아이콘: +
- 배경: Primary (#00C73C)
- 그림자: shadow-lg
- 클릭: 할일 추가 모달 열기

**빈 상태**:
```

│             📝                       │
│                                      │

│                                      │

```

**로딩 상태**:
- 중앙 스피너
- 반투명 오버레이

## 4. 상호작용 패턴

### 4.1 할일 완료 처리

**데스크톱**:
1. 사용자가 체크박스 클릭
2. 즉시 체크 표시 + 초록색 배경 애니메이션 (0.3s)
3. 항목이 회색으로 변하며 취소선 표시
4. 목록 재정렬 (완료된 항목 하단 이동)
5. Toast 알림: "할일을 완료했습니다"

**모바일**:
- 동일하게 체크박스 터치

### 4.2 할일 삭제

**데스크톱**:
1. 사용자가 삭제 버튼 (🗑️) 클릭
2. 확인 다이얼로그 표시:
   ```
   ┌──────────────────────────────┐
   │  할일 삭제                   │
   ├──────────────────────────────┤
   │  이 할일을 휴지통으로        │
   │  이동하시겠습니까?           │
   │                              │
   │    [취소]  [삭제]            │
   └──────────────────────────────┘
   ```
3. "삭제" 클릭 시:
   - 항목이 페이드아웃 (0.3s)
   - 목록에서 제거
   - Toast 알림: "할일이 휴지통으로 이동되었습니다"

**모바일**:
- 선택 사항: 좌측 스와이프로 삭제 액션 표시
  ```
  ┌──────────────────────────────────────┐
  │ ← [삭제]  프로젝트 마감              │
  └──────────────────────────────────────┘
  ```
- 또는 삭제 버튼 터치

### 4.3 모달 열기/닫기

**열기 애니메이션**:
- 오버레이: 페이드인 (0.2s)
- 모달 컨테이너: 스케일업 + 페이드인 (0.3s)

**닫기**:
- ESC 키
- 오버레이 클릭
- 닫기 버튼 (×) 클릭
- 취소 버튼 클릭

**닫기 애니메이션**:
- 오버레이: 페이드아웃 (0.2s)
- 모달 컨테이너: 스케일다운 + 페이드아웃 (0.3s)

### 4.4 필터 적용

1. 사용자가 필터 버튼 클릭 (예: "진행중")
2. 버튼 스타일 변경 (Primary 배경)
3. 로딩 스피너 표시 (0.5s)
4. 필터링된 목록 표시
5. 결과 개수 표시: "진행중인 할일 3개"

### 4.5 검색

1. 사용자가 검색 입력 필드에 타이핑
2. Debounce 300ms 후 검색 실행
3. 로딩 인디케이터 (입력 필드 우측 스피너)
4. 검색 결과 표시
5. 결과 없음:
   ```
   ┌──────────────────────────────────────┐
   │             🔍                       │
   │                                      │
   │      검색 결과가 없습니다            │
   │                                      │
   │      다른 키워드로 검색해보세요      │
   │                                      │
   └──────────────────────────────────────┘
   ```

### 7.4 접근성 체크리스트

**키보드 네비게이션**:
- [ ] Tab으로 모든 인터랙티브 요소 접근 가능
- [ ] Enter로 버튼 활성화
- [ ] ESC로 모달 닫기
- [ ] 화살표 키로 드롭다운 네비게이션

**ARIA 속성**:
- [ ] 버튼: `aria-label` (아이콘 버튼)
- [ ] 모달: `role="dialog"`, `aria-modal="true"`
- [ ] 입력 필드: `aria-invalid`, `aria-describedby` (에러 시)
- [ ] 로딩: `aria-label="로딩 중"`

**포커스 인디케이터**:
- [ ] 모든 포커스 가능한 요소에 명확한 outline
- [ ] 색상 대비 4.5:1 이상

**스크린 리더**:
- [ ] 의미 있는 HTML 태그 사용 (semantic HTML)
- [ ] 대체 텍스트 제공 (이미지, 아이콘)

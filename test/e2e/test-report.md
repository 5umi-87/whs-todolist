## 테스트 시나리오 및 결과

### 1. 시나리오 3.1.1: 신규 사용자 회원가입 ✅

**목표**: 신규 사용자 계정 생성 및 자동 로그인

**테스트 단계**:
1. 회원가입 페이지 접속
2. 사용자 정보 입력
   - 이메일: test_e2e_user@example.com
   - 이름: E2E테스트유저
   - 비밀번호: TestPassword123!
3. 회원가입 버튼 클릭

**결과**: ✅ 성공
- 회원가입 완료 후 로그인 페이지로 자동 리다이렉트
- 계정이 정상적으로 생성됨

---

### 2. 시나리오 3.1.2: 기존 사용자 로그인 ✅

**목표**: 생성한 계정으로 로그인하여 메인 화면 진입

**테스트 단계**:
1. 로그인 정보 입력
   - 이메일: test_e2e_user@example.com
   - 비밀번호: TestPassword123!
2. 로그인 버튼 클릭

**결과**: ✅ 성공
- JWT 토큰 발급 및 자동 로그인 상태 유지
- 메인 할일 목록 페이지로 이동
- 사용자 이름 "E2E테스트유저" 상단에 표시

---

### 3. 시나리오 2.1.1: 할일 추가 ✅

**목표**: 새로운 할일 2개 추가

**테스트 단계**:
1. **첫 번째 할일 추가**
   - 제목: "팀장님께 보고서 제출"
   - 내용: "오전 11시까지 제출 필요"
   - 시작일: 2025-11-28
   - 마감일: 2025-11-28

2. **두 번째 할일 추가**
   - 제목: "주간 보고서 작성"
   - 내용: "금주 업무 정리 및 다음주 계획"
   - 시작일: 2025-11-28
   - 마감일: 2025-11-30

**결과**: ✅ 성공
- 두 개의 할일이 성공적으로 추가됨
- 마감일에 따라 자동으로 D-day 계산 표시
  - "팀장님께 보고서 제출": ⚠️ D+1
  - "주간 보고서 작성": ⚠️ D-3
- 상태가 "진행중"으로 표시됨

---

### 4. 시나리오 2.1.2: 할일 완료 처리 ✅

**목표**: 할일 완료 체크 및 상태 변경

**테스트 단계**:
1. "팀장님께 보고서 제출" 할일의 완료 버튼 클릭

**결과**: ✅ 성공
- 할일 상태가 "완료"로 변경됨
- 완료 버튼이 "미완료로 변경" 버튼으로 전환
- 수정 버튼이 숨겨지고 삭제 버튼만 표시됨

---

### 5. 시나리오 2.1.3: 삭제한 할일 복원 ✅

**목표**: 실수로 삭제한 할일을 휴지통에서 복원

**테스트 단계**:
1. "주간 보고서 작성" 할일 삭제
2. 휴지통 메뉴로 이동
3. 삭제된 할일 확인
4. 복원 버튼 클릭
5. 할일 목록으로 돌아가서 확인

**결과**: ✅ 성공
- 할일이 휴지통으로 이동됨
- 삭제 시간 표시: "5초 전"
- 복원 버튼 클릭 후 원래 할일 목록에 다시 표시됨
- 원래 상태(진행중) 그대로 유지됨

---

### 6. 시나리오 2.1.4: 국경일 조회 ✅

**목표**: 2025년 국경일 목록 확인

**테스트 단계**:
1. 국경일 메뉴 클릭
2. 2025년 국경일 목록 확인

**결과**: ✅ 성공
- 2025년 국경일이 정상적으로 표시됨
- 총 15개 국경일 확인
  - 신정 (01-01)
  - 설날 연휴 (01-28, 01-29, 01-30)
  - 삼일절 (03-01)
  - 어린이날 (05-05)
  - 부처님오신날 (05-05)
  - 현충일 (06-06)
  - 광복절 (08-15)
  - 개천절 (10-03)
  - 추석 연휴 (10-05, 10-06, 10-07)
  - 한글날 (10-09)
  - 성탄절 (12-25)
- 각 국경일에 이모지와 설명이 표시됨
- "매년 반복" 태그 표시
